### 🟢 Backend Reception (Node.js/Express)

```javascript
app.post('/api/register', async (req, res) => {
  // 1. Validate required fields
  const { fullName, email, phone, password } = req.body;
  if (!fullName || !email || !phone || !password) {
    return res.status(400).json({ error: 'All fields required' });
  }

  // 2. Normalize email
  const emailLc = email.trim().toLowerCase();

  // 3. Duplicate Check (Firestore Query)
  let existing = await db.collection('registrations')
    .where('email_lc', '==', emailLc)
    .limit(1)
    .get();

  // Backward compatibility: fallback to non-normalized email
  if (existing.empty) {
    existing = await db.collection('registrations')
      .where('email', '==', email.trim())
      .limit(1)
      .get();
  }

  if (!existing.empty) {
    return res.status(409).json({ 
      error: 'This email is already registered. Please log in.' 
    });
  }

  // 4. Firestore Insert
  const ref = await db.collection('registrations').add({
    full_name: fullName,
    email: email.trim(),
    email_lc: emailLc,
    phone: phone,
    password: password,
    registration_status: 'step1_complete',
    created_at: new Date().toISOString(),
    updated_at: new Date().toISOString()
  });

  const userId = ref.id; // ✅ Correct: Using ref.id

  // 5. Trigger Email Notifications (Non-blocking)
  try {
    const { sendAdminEmail, sendConfirmationEmail } = require('./services/emailService');

    // Send admin notification
    await sendAdminEmail({
      userId,
      fullName,
      email,
      phone
    }, 'New Registration Lead - Step 1');

    // Send user confirmation (skip if admin address)
    const adminList = [...new Set(
      [process.env.ADMIN_EMAIL, ...process.env.ADMIN_EMAILS.split(',')]
        .map(e => e.trim())
        .filter(Boolean)
    )];
    
    if (!adminList.includes(email.trim())) {
      await sendConfirmationEmail(email, fullName);
    }

  } catch (emailError) {
    console.error('Email error (signup):', emailError);
    // ✅ Non-blocking: Registration succeeds even if email fails
  }

  // 6. Return Success Response
  return res.status(201).json({
    success: true,
    message: 'Registration successful!',
    userId: userId
  });
});
```

## 2. Backend Logic: `upsert_registration` Function

**File:** [public_html/api/index.php](public_html/api/index.php#L192)  
**Function Type:** PHP (Alternative backend for PHP-based endpoints)

### Function Signature & Logic

```php
function upsert_registration(array $data, bool $isUpdate = false): array
{
    // 1. Email Normalization
    $email = normalize_email((string) ($data['email'] ?? ''));

    // 2. Build Record Payload
    $record = [
        'full_name' => (string) ($data['fullName'] ?? $data['full_name'] ?? ''),
        'email' => (string) ($data['email'] ?? ''),
        'email_lc' => $email,  // ✅ Lowercase for case-insensitive queries
        'phone' => (string) ($data['phone'] ?? ''),
        'password' => (string) ($data['password'] ?? ''),
        'service' => (string) ($data['service'] ?? $data['preferredCountry'] ?? ''),
        'preferred_country' => (string) ($data['preferredCountry'] ?? $data['preferred_country'] ?? ''),
        'source' => (string) ($data['source'] ?? 'Website'),
        'registration_status' => (string) ($data['registration_status'] ?? 'step1_completed'),
        'updated_at' => now_iso(),
        'created_at' => (string) ($data['created_at'] ?? now_iso()),
    ];

    // 3. Determine Document ID Strategy
    $documentId = '';
    
    if ($isUpdate && !empty($data['userId'])) {
        // Use provided userId for updates
        $documentId = trim((string) $data['userId']);
    }
    
    if ($documentId === '') {
        // Fallback to email as identifier
        $documentId = $email;
    }
    
    if ($documentId === '') {
        // Last resort: random bytes
        $documentId = bin2hex(random_bytes(8));
    }

    // 4. Firestore Write
    return firestore_set_document('registrations', $documentId, $record);
}
```

### Data Integrity & Field Cleaning

**Function:** `firestore_document_payload()` (Line 262)

```php
function firestore_document_payload(array $data): array
{
    $clean = [];

    foreach ($data as $key => $value) {
        $key = (string) $key;
        
        // ✅ Filter OUT system fields to prevent Firestore metadata pollution
        if ($key === 'id' || $key === '__name__' || str_starts_with($key, '__')) {
            continue;  // Skip system fields
        }

        $clean[$key] = $value;
    }

    return $clean;
}
```

**Why Remove `__name__`?**
- Firestore auto-generates metadata fields like `__name__` when documents are retrieved
- These are **read-only** system fields and cannot be re-written to Firestore
- Including them in upserts would cause write failures
- The filter prevents this by excluding all fields starting with `__`

### Email Normalization

```php
function normalize_email(string $email): string
{
    return strtolower(trim($email));
}
```

**Purpose:** Ensures case-insensitive email queries (e.g., `John@Example.com` == `john@example.com`)

### Core Brevo Send Function

```javascript
const sendBrevoEmail = async ({ to, subject, html }) => {
  const apiKey = getEnvString('BREVO_API_KEY');
  const senderEmail = getEnvString('BREVO_SENDER_EMAIL') || 'info@avcareerz.com';
  const senderName = getEnvString('BREVO_SENDER_NAME') || 'avcareerz';

  if (!apiKey || !senderEmail) {
    throw new Error('Missing BREVO_API_KEY or BREVO_SENDER_EMAIL');
  }

  const payload = {
    sender: { 
      email: senderEmail, 
      name: senderName 
    },
    to: Array.isArray(to)
      ? to.map((email) => ({ email }))
      : [{ email: to }],
    subject,
    htmlContent: html
  };

  try {
    const response = await axios.post(
      'https://api.brevo.com/v3/smtp/email',
      payload,
      {
        headers: {
          'api-key': process.env.BREVO_API_KEY,
          'Content-Type': 'application/json',
          'Accept': 'application/json'
        },
        timeout: 15000  // 15-second timeout
      }
    );

    return response.data;  // Returns { messageId, id } on success

  } catch (err) {
    const status = err?.response?.status;
    const data = err?.response?.data;
    
    // Log full error for diagnostics
    console.error('Brevo send error:', {
      status,
      data,
      message: err?.message
    });
    
    throw err;  // ⚠️ Will interrupt registration if not caught
  }
};
```

### Critical Issues & Bottlenecks

#### ⚠️ Issue #1: API Key Exposure (CRITICAL SECURITY)
- **Location:** [includes/.env](includes/.env#L7)
- **Problem:** Brevo API key is hardcoded in plaintext
- **Risk:** Anyone with access to codebase can send emails on your account
- **Recommendation:**
  - Move `.env` to `.gitignore`
  - Use environment secrets in production (Render, Vercel, etc.)
  - Rotate the API key immediately

#### ⚠️ Issue #2: Non-Blocking Email Failures
- **Location:** [includes/server.js](includes/server.js#L254-L262)
- **Problem:** Email service failures don't block registration but user doesn't know
- **Impact:** Users get success but never receive confirmation email
- **Symptom:** Form says "success" but no email arrives
- **Recommendation:** Implement retry logic or queue emails separately

#### ⚠️ Issue #3: No Email Sent Verification
- **Location:** [includes/emailService.js](includes/emailService.js)
- **Problem:** No webhook listener to confirm email delivery
- **Impact:** Can't verify if Brevo actually sent the email
- **Recommendation:** Enable Brevo webhooks for delivery status

#### ⚠️ Issue #4: Password Stored in Plaintext
- **Location:** [includes/server.js](includes/server.js#L234)
- **Problem:** Password is stored as-is in Firestore
- **Risk:** High - password compromise if database breached
- **Recommendation:** Hash passwords using bcrypt or similar

#### ⚠️ Issue #5: 15-Second Email Timeout
- **Location:** [includes/emailService.js](includes/emailService.js#L48)
- **Problem:** Long Brevo API latency could exceed timeout
- **Impact:** Registration appears successful but email never sends
- **Recommendation:** Increase timeout to 30s or use queue system

#### ✅ Issue #6: No Duplicate Email Validation (Frontend)
- **Location:** [Frontend/next-form.html](Frontend/next-form.html)
- **Problem:** Frontend doesn't check email availability before submission
- **Impact:** User submits form and only then gets "email exists" error
- **Recommendation:** Add async email validation on blur

#### ✅ Issue #7: Firestore Backward Compatibility
- **Location:** [includes/server.js](includes/server.js#L235-L242)
- **Status:** ✅ Good - has fallback for non-normalized emails

#### ⚠️ Issue #8: No Rate Limiting
- **Location:** [includes/server.js](includes/server.js#L221)
- **Problem:** No rate limiting on `/api/register`
- **Risk:** Spam/brute-force registrations possible
- **Recommendation:** Add IP-based or email-based rate limiting

#### ⚠️ Issue #9: WhatsApp Notification Error Handling
- **Location:** [includes/server.js](includes/server.js#L269-L273)
- **Problem:** WhatsApp errors are warnings but could fail silently
- **Recommendation:** Add proper logging/monitoring
